context: 'installOneHook composes our POSIX-sh block (marker-guarded) into an existing hook by APPENDING it, regardless of the existing file''s shebang. If that hook uses a non-sh interpreter (#!/usr/bin/env python, a Node/Husky dispatcher), our appended /bin/sh lines can break it or never run. Also our block runs LAST, so an existing hook that exits early (explicit exit, or set -e failure) silently skips our shim (current-quest injection, commit-msg require_quest enforcement, post-commit linking). Fix direction: detect the existing shebang; if it is not sh/bash, warn and skip (or install a side-by-side companion) rather than corrupting it; consider documenting the run-last ordering caveat.'
