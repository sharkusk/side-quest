--- note 2026-07-04T22:22:06Z ---

Scope expansion (agreed): fold the "onboard is the single front-door command" simplification into this quest. Because onboard becomes plugin-aware (detects the plugin → skips .mcp.json; no plugin → writes it) and is idempotent, it becomes the ONE per-repo command for both distribution paths AND for both first-setup and refresh — collapsing the old "plugin users run init+install-hooks, manual users run onboard" split into a single instruction. init/install-hooks stay as lower-level subcommands onboard composes (make dev still calls install-hooks in the dev loop) but are demoted out of the front-door docs to an "advanced" --help grouping. Re-running onboard is the "update a repo" path (leaves existing ref/.mcp.json as-is, refreshes hooks vOLD→vNEW). Captured as decisions D6 + D9 in docs/superpowers/specs/2026-07-04-plugin-cli-provisioning-and-lifecycle-design.md.

--- note 2026-07-04T22:24:30Z ---

Refinement: the plugin-detected case should SILENTLY skip writing .mcp.json — no prompt, no "skipping .mcp.json" message. (Supersedes the "prompt before writing" framing in the title.) .mcp.json is internal plumbing; announcing the skip would only confuse an end user who doesn't know the file exists. onboard just does the right thing quietly. Captured in D6 of the spec.
