--- note 2026-07-06T01:51:31Z ---

Confirmed on Windows: Claude wipes NOTHING on uninstall — the persistent data dir AND every cached plugin version (0.1.4/0.1.5/0.1.6) all survive. So there is no reliable signal (neither data-dir nor cache-dir absence) for the launcher's "plugin gone -> self-remove" heuristic to key on; that heuristic is dead code and can never fire.

Fix direction (revises D3): teardown must be explicit. Make `uninstall-cli` (or a new `side-quest uninstall`) remove side-quest's OWN footprint — the PATH launcher AND the provisioned ${CLAUDE_PLUGIN_DATA}/bin/side-quest.exe (+ marker). Cleaning up the binary we provisioned is within our footprint (branch-safety allows it); the "launcher never writes the data dir" rule only held under the false auto-wipe premise. It can run via the orphaned launcher even after the plugin is removed. Also retire the dead data-dir-absent self-removal branch in launcher.sh/.cmd, and document that plugin uninstall leaves our binary+launcher behind with `side-quest uninstall-cli` as the one-command teardown. Do NOT touch Claude's plugin cache dir (Claude-managed).

--- note 2026-07-06T02:27:51Z ---

CORRECTION from user (2026-07-06): the self-removal is NOT dead everywhere. On Linux/macOS, uninstalling the plugin via Claude Code DOES wipe the data dir, so the launcher correctly detects it's inert and offers removal — that feature works and must be KEPT. The failure is WINDOWS-ONLY: there Claude leaves the data dir (and all cached versions) in place, so the data-dir-absent heuristic never fires. Revises the earlier plan: do NOT retire the self-removal branch. Scope the fix to Windows — since Claude doesn't auto-wipe there, add explicit teardown (uninstall-cli also removes the provisioned side-quest.exe) while keeping the auto-detect self-removal that already works on posix. Net: behavior is intentionally platform-split — auto on Linux/macOS, explicit on Windows.

--- note 2026-07-06T03:42:26Z ---

RESOLVED: document-and-close (user decision 2026-07-06). Since the launcher self-removal WORKS on macOS/Linux (Claude wipes the data dir there) and only Windows leaves the binary+launcher behind, we chose to document the platform split rather than build a Windows-only teardown. Documented in docs/plugin.md new "Uninstalling" section (commit c7feab5, merged to main): macOS/Linux auto-clean + self-removing launcher; Windows leaves the data dir, so run `side-quest uninstall-cli` and delete %USERPROFILE%\.claude\plugins\data\side-quest-side-quest manually; quests are safe either way (they live in git refs, not the plugin). No code change; not release-gated.
