--- note 2026-07-06T01:45:08Z ---

Confirmed on Windows during v0.1.6 rollout: an install path reported the CLI "already installed" and did NOT refresh the stale 0.1.5 side-quest.cmd; the user had to manually delete the stale .cmd before install worked. Note cli.Install() (the `install-cli` subcommand) DOES overwrite a marked launcher — so the culprit is a status-gated path (the MCP cli_install tool and/or the agent checking cli_status first) that short-circuits on installed==true instead of refreshing. Fix direction: make the install path always (re)write the launcher body when the existing file is marked (a refresh), and/or have cli_install overwrite rather than skip when already-installed.

--- note 2026-07-06T02:02:27Z ---

FIX PLAN (self-healing refresh, no agent dependency). Root cause confirmed by reading code: cli.Install() DOES overwrite a marked launcher, and the cli_install MCP tool calls it unconditionally — so the code would refresh. The gap is cli_status only returns Installed bool (no staleness), so the agent sees installed==true and never re-runs install. And a stale 0.1.5 launcher can't self-update by running: it fails "binary not found" (globs side-quest-*) BEFORE reaching the new fixed-name binary. So the refresh must come FROM the new binary.

Implement:
1. internal/cli: add Refresh() []string — for each dir in launcherDirs(), take dir/LauncherName(); if isMarkedLauncher and its bytes != LauncherBody(), os.WriteFile(body,0o755) and record the path. Only marked files, only the OS LauncherName(), only when stale. (bytes import.)
2. cmd/side-quest/serve.go: at serve startup call cli.Refresh() best-effort and log each refreshed path to stderr ("side-quest: refreshed stale CLI launcher at %s"). serve runs every session, so an upgraded binary self-heals a stale launcher on the first connect — no agent action, no re-offer (only rewrites an already-consented marked launcher we own; stdout MCP protocol untouched since we log to stderr).
3. Tests (internal/cli/launcher_test.go): stale marked launcher -> rewritten to LauncherBody()+returned; current content -> untouched+empty; unmarked file -> untouched.
Ships in 0.1.7. Prevents recurrence; the 0.1.6 stale launcher is a one-time manual fix (already done). Optional add-on: Stale bool on cli_status for visibility, and update SKILL/guidance so the agent offers a refresh when stale.
