--- note 2026-07-04T19:38:53Z ---

Finding: adding `name: side-quest` to commands/sq.md frontmatter does NOT relabel the menu entry — it still shows "sq". The docs say `name:` sets the display label (defaulting to the filename), but the current Claude Code version doesn't honor it for slash commands; a command's invocation id is always its filename. Verified: /reload-plugins + /reload-skills + /skills all report "no changes". The description IS read from the frontmatter, but the label falls back to the filename `sq`.

Decision: leave it as `sq` for now. Keeping the `name: side-quest` frontmatter (and TestSqCommandDisplayName) — it's spec-correct and forward-compatible, so it'll take effect if a future version honors `name:` for commands. The only way to force the "side-quest" label today is to rename the file to commands/side-quest.md, which changes the trigger from /sq to /side-quest — not worth losing the short trigger. Revisit if the label matters more later, or if a Claude Code update starts honoring the field.

--- note 2026-07-04T19:43:04Z ---

Plugin-scope angle (refines the decision): the "sq" label is because this repo dogfoods via a project symlink (make dev → .claude/commands/sq.md), which renders as "sq project" — bare filename, no namespace. Scope-namespacing is why superpowers shows "superpowers:brainstorming plugin".

Per Claude Code docs (verified via claude-code-guide): plugin SKILLS (skills/<name>/SKILL.md) are definitively namespaced as <plugin>:<skill> and require the full form. Plugin COMMANDS (commands/*.md — the legacy layout; docs say "use skills/ for new plugins") are a DOCUMENTATION GAP: namespacing/display not specified. So we can't assert from docs whether the shipped commands/sq.md renders as "side-quest:sq" or bare "sq" under a plugin install.

The side-quest plugin is named "side-quest" (.claude-plugin/plugin.json), so IF plugin commands namespace like skills, real plugin users already see "side-quest:sq" for free — making this quest moot for the actual distribution channel, with only the dev-repo symlink showing bare "sq".

Empirical test to settle it (nobody has run it yet): install side-quest from its local marketplace (.claude-plugin/marketplace.json, source ".") and look at the menu — does it show "side-quest:sq"? If yes → close as already-satisfied-by-plugin, no code change. If it shows bare "sq" even as a plugin → the only lever is surfacing capture as a skill in skills/ (namespaced) or renaming the command. Staying deferred until that check is run.
