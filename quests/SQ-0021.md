context: 'The hook shims embed the absolute os.Executable() path verbatim (hooks.go:29-46) with no forward-slash normalization. On Windows that is a backslash drive path (C:\Users\...\side-quest.exe); Git runs hooks via Git-for-Windows MSYS sh, where backslashes are escapes and drive-letter paths are fragile, so the shim may fail to invoke the binary. Fix: filepath.ToSlash (or an MSYS /c/... form) when composing the shim, and actually verify on Git for Windows that the extensionless sh hooks run + call the .exe. Ties into the Phase 7 unverified-Windows note.'
