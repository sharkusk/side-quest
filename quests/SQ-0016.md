context: 'addRefspec sets remote.origin.push=refs/side-quest/*:refs/side-quest/*. Because a configured remote.<name>.push becomes the ENTIRE default refspec (push.default is bypassed), a bare ''git push'' then pushes ONLY the quest refs and NOT the user''s current branch — verified via isolated dry-run. Users would think they pushed code but only quests went up. Quest ref auto-push is desirable; breaking branch push is not. Fix options: drop the push refspec (rely on explicit push / planned sync command) OR make it additive so the branch still pushes. Found when user asked whether the quest ref auto-pushes for regular users.'

README: document how quest data syncs across machines. install-hooks adds a FETCH refspec, so git fetch/pull auto-pull refs/side-quest/quests (fetch refspecs are additive — verified). BUT a fresh git clone does NOT include the quest ref (custom refs aren't fetched by default) — run install-hooks first, then fetch. Push side: once this bug is fixed, document that install-hooks configures push too; until then share quests via explicit 'git push origin refs/side-quest/quests'. Add a short 'syncing quests across machines' section to the README install/remote docs.
