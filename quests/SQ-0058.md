--- note 2026-07-04T16:15:16Z ---

Cross-project evidence (2026-07-04): the .gitignore shim block is HAND-maintained — side-quest writes no .gitignore at all. The user's OTHER project already carries the full four-entry block including /.githooks/pre-push and the "these four" comment, copied from side-quest's own block and manually corrected, while side-quest's own repo still had the three-entry SQ-0057 bug until today. So the dogfood repo lagged the user's manual fix, and the user has been patching this per-project by hand each time a shim appears. The hand-written marker-style comment block is effectively a prototype of the feature: install-hooks should write/refresh a marker-guarded, version-aware gitignore block listing its shims whenever the resolved hooks dir is inside the work tree (tracked hooksPath). Scope = the FULL auto-managed block (warning-only is insufficient — user is still copy-pasting). Since the user habitually uses a tracked .githooks layout, this is their normal case, not an edge case → recommend launch=0.1.0.
